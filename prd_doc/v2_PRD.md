# PRD 增补：新增「图形记忆测试」与「位置记忆测试」

> 本段 PRD 用于在现有认知训练小游戏网站中新增两个游戏模块：**图形记忆测试**与**位置记忆测试**。Codex 实现时应复用现有游戏框架，包括首页游戏卡片、游戏页布局、起始难度选择器、结果弹窗、本地最佳成绩存储与通用状态机。

---

## 1. 新增游戏概述

### 1.1 新增游戏列表

| 游戏 | 路径建议 | 核心训练能力 | 一句话描述 |
|---|---|---|---|
| 图形记忆测试 | `/games/shape-memory` | 视觉记忆、图形识别、短时记忆 | 用户短暂记住一组图形及其特征，随后从候选项中识别或复原。 |
| 位置记忆测试 | `/games/position-memory` | 空间记忆、视觉定位、注意力 | 用户短暂记住物体在网格中的位置，隐藏后点击原位置。 |

### 1.2 通用要求

1. 两个新游戏都必须支持**人工选择起始关卡难度**。
2. 两个新游戏都必须接入现有首页游戏卡片列表。
3. 两个新游戏都必须接入现有最佳成绩存储逻辑。
4. 两个新游戏都必须复用现有 `GameHeader`、`DifficultySelector`、`ResultModal`、`GridBoard` 等通用组件；如组件能力不足，可在保持兼容的前提下扩展。
5. 两个新游戏都应支持桌面端和移动端。
6. 游戏展示阶段应禁用用户输入，避免提前点击造成状态错乱。

## 2. 游戏五：图形记忆测试

### 2.1 游戏目标

图形记忆测试用于训练用户的**视觉短时记忆、图形识别能力、颜色与形状组合记忆能力**。系统会短暂展示一组图形，随后隐藏图形，并让用户从候选项中选择刚才出现过的图形。

### 2.2 核心玩法

#### 基础模式：识别出现过的图形

1. 当前关卡生成 `N` 个目标图形。
2. 每个图形由以下属性组成：
   - `shape`：形状，例如圆形、方形、三角形、星形、菱形、六边形。
   - `color`：颜色，例如红、蓝、绿、黄、紫、橙。
   - `pattern`：可选纹理，例如纯色、条纹、点状、空心。
3. 系统将目标图形展示 `previewDuration` 毫秒。
4. 展示结束后目标图形隐藏。
5. 系统展示一组候选图形，其中包含刚才出现过的目标图形，也包含干扰图形。
6. 用户需要选中所有刚才出现过的图形。
7. 用户选中全部目标且没有选错，则当前关卡通过。
8. 用户点击任一干扰图形，则当前关卡失败。

### 2.3 起始难度配置

| 参数 | 说明 | 建议值 |
|---|---|---|
| `startLevel` | 起始关卡 | 默认 1，可选 1 到 20 |
| `targetCount` | 目标图形数量 | 随关卡递增 |
| `candidateCount` | 候选图形总数 | 始终大于目标图形数量 |
| `previewDuration` | 目标图形展示时长 | 随关卡递减 |
| `shapePoolSize` | 可用形状数量 | 随关卡递增 |
| `colorPoolSize` | 可用颜色数量 | 随关卡递增 |
| `usePattern` | 是否启用纹理维度 | 中高难度开启 |

### 2.4 难度曲线建议

| 关卡范围 | 目标图形数 | 候选图形数 | 形状池 | 颜色池 | 纹理 | 展示时长 |
|---|---:|---:|---:|---:|---|---:|
| 1-3 | 2 | 4 | 3 | 3 | 不启用 | 1800ms |
| 4-6 | 3 | 6 | 4 | 4 | 不启用 | 1700ms |
| 7-10 | 4 | 8 | 5 | 5 | 不启用 | 1600ms |
| 11-15 | 5 | 10 | 6 | 6 | 启用 | 1500ms |
| 16-20 | 6 | 12 | 6 | 6 | 启用 | 1400ms |

### 2.5 交互流程

1. 用户进入图形记忆测试页。
2. 页面展示游戏说明、历史最佳成绩、起始关卡选择器。
3. 用户选择起始关卡并点击「开始」。
4. 系统根据关卡生成目标图形。
5. 页面居中展示目标图形，同时展示倒计时进度条。
6. 展示结束后，目标图形消失。
7. 系统展示候选图形列表。
8. 用户点击自己记住的图形。
9. 每次点击后，被选中的图形显示选中态。
10. 当用户选中数量达到目标数量时，系统自动判定结果。
11. 如果全部正确，进入下一关。
12. 如果包含错误图形，游戏失败并展示结果弹窗。

### 2.6 判定规则

| 用户行为 | 系统判定 |
|---|---|
| 点击目标图形 | 标记为已选中 |
| 点击干扰图形 | 立即失败，或在选满后统一判错；首期建议立即失败 |
| 选中全部目标图形 | 当前关卡通过 |
| 重复点击已选图形 | 取消选中或无效；首期建议无效，降低误操作风险 |
| 展示阶段点击 | 无效 |

### 2.7 结果反馈

结果弹窗展示：

- 本轮达到的最高关卡
- 本轮最高目标图形数
- 是否刷新最佳成绩
- 失败关卡的正确图形
- 用户错误选择的图形，若有
- 操作按钮：
  - 再玩一次
  - 调整起始难度
  - 返回首页

### 2.8 数据模型建议

```typescript
type ShapeType = 'circle' | 'square' | 'triangle' | 'star' | 'diamond' | 'hexagon';
type ShapeColor = 'red' | 'blue' | 'green' | 'yellow' | 'purple' | 'orange';
type ShapePattern = 'solid' | 'outline' | 'striped' | 'dotted';

type ShapeItem = {
  id: string;
  shape: ShapeType;
  color: ShapeColor;
  pattern: ShapePattern;
};

type ShapeMemoryState = {
  level: number;
  targets: ShapeItem[];
  candidates: ShapeItem[];
  selectedIds: string[];
  status: GameStatus;
  previewDuration: number;
};
```

### 2.9 实现提示

1. 图形可优先使用 CSS 实现，不必引入图片素材。
2. 每个 `ShapeItem` 的组合应尽量唯一，避免同一关中出现完全相同的图形。
3. 干扰图形应与目标图形相似但不完全相同，例如同形不同色、同色不同形。
4. 移动端候选图形建议使用 2 到 3 列网格。
5. 图形组件建议抽象为 `ShapeTile`，便于复用。

## 3. 游戏六：位置记忆测试

### 3.1 游戏目标

位置记忆测试用于训练用户的**空间记忆、位置编码能力、视觉定位能力**。系统会在网格中短暂展示若干目标物体，随后隐藏物体，用户需要点击这些物体原本所在的位置。

### 3.2 核心玩法

1. 当前关卡生成一个 `rows × cols` 的网格。
2. 系统随机选择 `N` 个格子作为目标位置。
3. 展示阶段，这些目标格子中显示物体或高亮标记。
4. 展示 `previewDuration` 毫秒后，所有目标隐藏。
5. 用户点击刚才出现过目标的格子。
6. 点击正确位置则标记为已找到。
7. 点击错误位置则失败。
8. 找到全部目标位置后，当前关卡通过并进入下一关。

### 3.3 与视觉记忆测试的区别

现有视觉记忆测试通常是“记住亮起的格子”。位置记忆测试应更强调“物体在空间中的位置”，可以在格子中展示不同物体图标，但首期可以先使用统一图标实现。

| 对比项 | 视觉记忆测试 | 位置记忆测试 |
|---|---|---|
| 记忆对象 | 高亮格子 | 物体或标记所在位置 |
| 主要训练点 | 视觉瞬时记忆 | 空间定位与位置编码 |
| 展示形式 | 格子亮灭 | 物体出现后消失 |
| 扩展方向 | 更多格子、更短时间 | 不同物体、位置与物体绑定 |

### 3.4 起始难度配置

| 参数 | 说明 | 建议值 |
|---|---|---|
| `startLevel` | 起始关卡 | 默认 1，可选 1 到 20 |
| `gridRows` | 网格行数 | 随关卡递增 |
| `gridCols` | 网格列数 | 随关卡递增 |
| `targetCount` | 目标位置数量 | 随关卡递增 |
| `previewDuration` | 目标展示时长 | 随关卡递减 |
| `allowWrongClick` | 是否允许错误点击 | 首期为 `false`，点错即失败 |

### 3.5 难度曲线建议

| 关卡范围 | 网格规模 | 目标位置数 | 展示时长 |
|---|---|---:|---:|
| 1-3 | 3 × 3 | 2-3 | 1800ms |
| 4-6 | 4 × 4 | 3-4 | 1700ms |
| 7-10 | 5 × 5 | 4-6 | 1600ms |
| 11-15 | 6 × 6 | 6-8 | 1500ms |
| 16-20 | 7 × 7 | 8-10 | 1400ms |

### 3.6 交互流程

1. 用户进入位置记忆测试页。
2. 页面展示游戏说明、历史最佳成绩、起始关卡选择器。
3. 用户选择起始关卡并点击「开始」。
4. 系统根据关卡生成网格和目标位置。
5. 展示阶段，目标位置出现物体图标或高亮标记。
6. 展示结束后，目标物体隐藏，网格恢复为空白状态。
7. 用户点击记住的目标位置。
8. 点击正确则该格子显示正确状态。
9. 点击错误则游戏失败。
10. 用户找齐全部目标位置后，当前关卡通过。
11. 系统短暂展示成功反馈，并进入下一关。

### 3.7 判定规则

| 用户行为 | 系统判定 |
|---|---|
| 点击目标位置 | 标记正确，并显示已找到状态 |
| 点击非目标位置 | 当前轮失败 |
| 点击已找到位置 | 无效，不重复计数 |
| 找齐全部目标位置 | 当前关卡通过 |
| 展示阶段点击 | 无效 |

### 3.8 结果反馈

结果弹窗展示：

- 本轮达到的最高关卡
- 本轮最高目标位置数
- 最大网格规模
- 是否刷新最佳成绩
- 失败时可短暂显示正确位置
- 操作按钮：
  - 再玩一次
  - 调整起始难度
  - 返回首页

### 3.9 数据模型建议

```typescript
type GridPosition = {
  row: number;
  col: number;
};

type PositionMemoryState = {
  level: number;
  rows: number;
  cols: number;
  targets: GridPosition[];
  selected: GridPosition[];
  status: GameStatus;
  previewDuration: number;
};
```

### 3.10 实现提示

1. 位置生成时不能重复。
2. 网格尺寸应根据屏幕宽度自适应，移动端避免格子过小。
3. 推荐复用现有 `GridBoard` 组件，并增加以下能力：
   - 支持 `rows`
   - 支持 `cols`
   - 支持 `cellState`
   - 支持点击禁用状态
4. 目标展示阶段可以使用统一图标，例如圆点、星星、宝石、旗帜。
5. 首期不要求用户记住“哪个物体在哪”，只要求记住“哪些位置出现过物体”。
6. 后续可扩展高级模式：不同物体与位置绑定，用户不仅要点位置，还要匹配物体类型。

## 5. 验收标准

### 5.1 图形记忆测试验收

- [ ] 首页出现「图形记忆」游戏卡片。
- [ ] 点击卡片后进入 `/games/shape-memory`。
- [ ] 游戏开始前可以选择起始关卡。
- [ ] 选择不同起始关卡后，目标图形数量、候选图形数量、展示时长会相应变化。
- [ ] 展示阶段用户不能提前选择候选项。
- [ ] 候选项中必须包含全部目标图形和若干干扰图形。
- [ ] 用户选中全部目标图形且没有选错时，当前关卡通过。
- [ ] 用户点击干扰图形时，当前轮失败。
- [ ] 失败后展示结果弹窗。
- [ ] 通过关卡后自动进入下一关。
- [ ] 最佳成绩可保存到 `localStorage`。

### 5.2 位置记忆测试验收

- [ ] 首页出现「位置记忆」游戏卡片。
- [ ] 点击卡片后进入 `/games/position-memory`。
- [ ] 游戏开始前可以选择起始关卡。
- [ ] 选择不同起始关卡后，网格规模、目标数量、展示时长会相应变化。
- [ ] 展示阶段目标位置可见，展示结束后全部隐藏。
- [ ] 展示阶段用户点击无效。
- [ ] 用户点击目标位置后，该格子显示正确状态。
- [ ] 用户点击非目标位置时，当前轮失败。
- [ ] 用户找齐全部目标位置时，当前关卡通过。
- [ ] 失败后展示结果弹窗，并可选择再玩一次或调整难度。

### 5.3 通用边界验收

- [ ] 起始关卡低于最小值时，应修正为最小值或展示错误提示。
- [ ] 起始关卡高于最大值时，应修正为最大值或展示错误提示。
- [ ] 快速连续点击不会导致重复判定或状态错乱。
- [ ] 页面刷新后游戏回到初始说明状态。
- [ ] 移动端下所有格子和图形均可正常点击，且不出现横向滚动。
- [ ] 两个新增游戏不会影响原有四个游戏的路由、成绩存储和游戏流程。
