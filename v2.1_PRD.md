## 3. 游戏五：图形记忆测试

### 3.1 游戏目标

图形记忆测试用于训练用户的**视觉短时记忆、图形识别能力、颜色与形状组合记忆能力**。系统会短暂展示一组图形，随后隐藏图形，并让用户从候选项中选择刚才出现过的图形。

### 3.2 核心玩法

1. 当前关卡生成 `N` 个目标图形。
2. 每个图形由以下属性组成：
   - `shape`：形状，例如圆形、方形、三角形、星形、菱形、六边形。
   - `color`：颜色，例如红、蓝、绿、黄、紫、橙。
   - `pattern`：可选纹理，例如纯色、条纹、点状、空心。
3. 系统将目标图形展示 `previewDuration` 毫秒。
4. 展示结束后目标图形隐藏。
5. 系统展示一组候选图形，其中包含刚才出现过的目标图形，也包含干扰图形。
6. 用户需要选中所有刚才出现过的图形。
7. 用户选中全部目标且没有选错，则当前关卡通过。
8. 用户点击任一干扰图形，则当前关卡失败。

### 3.3 起始难度配置

| 参数 | 说明 | 建议值 |
|---|---|---|
| `startLevel` | 起始关卡 | 默认 1，可选 1 到 20 |
| `targetCount` | 目标图形数量 | 随关卡递增 |
| `candidateCount` | 候选图形总数 | 始终大于目标图形数量 |
| `previewDuration` | 目标图形展示时长 | 随关卡递减 |
| `shapePoolSize` | 可用形状数量 | 随关卡递增 |
| `colorPoolSize` | 可用颜色数量 | 随关卡递增 |
| `usePattern` | 是否启用纹理维度 | 中高难度开启 |

### 3.4 难度曲线建议

| 关卡范围 | 目标图形数 | 候选图形数 | 形状池 | 颜色池 | 纹理 | 展示时长 |
|---|---:|---:|---:|---:|---|---:|
| 1-3 | 2 | 4 | 3 | 3 | 不启用 | 1800ms |
| 4-6 | 3 | 6 | 4 | 4 | 不启用 | 1700ms |
| 7-10 | 4 | 8 | 5 | 5 | 不启用 | 1600ms |
| 11-15 | 5 | 10 | 6 | 6 | 启用 | 1500ms |
| 16-20 | 6 | 12 | 6 | 6 | 启用 | 1400ms |

### 3.5 交互流程

1. 用户进入图形记忆测试页。
2. 页面展示游戏说明、历史最佳成绩、起始关卡选择器。
3. 用户选择起始关卡并点击「开始」。
4. 系统根据关卡生成目标图形。
5. 页面居中展示目标图形，同时展示倒计时进度条。
6. 展示结束后，目标图形消失。
7. 系统展示候选图形列表。
8. 用户点击自己记住的图形。
9. 每次点击后，被选中的图形显示选中态。
10. 当用户选中数量达到目标数量时，系统自动判定结果。
11. 如果全部正确，进入下一关。
12. 如果包含错误图形，游戏失败并展示结果弹窗。

### 3.6 判定规则

| 用户行为 | 系统判定 |
|---|---|
| 点击目标图形 | 标记为已选中 |
| 点击干扰图形 | 当前轮失败，首期建议立即失败 |
| 选中全部目标图形 | 当前关卡通过 |
| 重复点击已选图形 | 无效，不取消选中 |
| 展示阶段点击 | 无效 |

### 3.7 数据模型建议

```typescript
type ShapeType = 'circle' | 'square' | 'triangle' | 'star' | 'diamond' | 'hexagon';
type ShapeColor = 'red' | 'blue' | 'green' | 'yellow' | 'purple' | 'orange';
type ShapePattern = 'solid' | 'outline' | 'striped' | 'dotted';

type ShapeItem = {
  id: string;
  shape: ShapeType;
  color: ShapeColor;
  pattern: ShapePattern;
};

type ShapeMemoryState = {
  level: number;
  targets: ShapeItem[];
  candidates: ShapeItem[];
  selectedIds: string[];
  status: GameStatus;
  previewDuration: number;
};
```

### 3.8 实现提示

1. 图形优先使用 CSS 实现，不必引入图片素材。
2. 每个 `ShapeItem` 的组合应唯一，避免同一关出现完全相同的图形。
3. 干扰图形应与目标图形相似但不完全相同，例如同形不同色、同色不同形。
4. 移动端候选图形建议使用 2 到 3 列网格。
5. 图形组件建议抽象为 `ShapeTile`，便于复用。

## 4. 游戏六：N-back 测试

### 4.1 游戏目标

N-back 测试用于训练用户的**工作记忆、持续注意力、信息持续更新能力与反应抑制能力**。系统会连续展示刺激项，用户需要判断当前刺激是否与前 `N` 轮刺激相同。

### 4.2 核心玩法

首期建议实现**视觉位置 N-back**：在 3 × 3 网格中，每一轮随机高亮一个格子，用户判断当前高亮位置是否与前 `N` 轮的位置相同。

1. 游戏开始前，用户选择起始难度。
2. 起始难度决定 `nLevel`，例如 1-back、2-back、3-back。
3. 系统按固定节奏连续展示若干轮刺激。
4. 每轮刺激为 3 × 3 网格中的一个高亮格子。
5. 用户需要在当前轮判断：当前高亮格子是否与前 `N` 轮高亮格子相同。
6. 如果相同，用户点击「匹配」按钮或按键盘 `Space`。
7. 如果不同，用户不点击，或点击「不匹配」按钮；首期建议提供「匹配 / 不匹配」两个按钮，降低规则理解成本。
8. 每轮结束后系统记录用户判断是否正确。
9. 完成指定轮数后展示结果，包括正确率、命中率、误报率、漏报数与得分。

### 4.3 游戏模式

首期只做一种模式，后续可扩展。

| 模式 | 是否首期实现 | 说明 |
|---|---|---|
| 位置 N-back | 是 | 记忆网格中高亮格子的位置。 |
| 数字 N-back | 否 | 连续展示数字，判断当前数字是否与前 N 轮相同。 |
| 图形 N-back | 否 | 连续展示图形，判断当前图形是否与前 N 轮相同。 |
| 双重 N-back | 否 | 同时判断位置与声音或图形，难度较高，后续再做。 |

### 4.4 起始难度配置

| 参数 | 说明 | 建议值 |
|---|---|---|
| `startLevel` | 起始关卡 | 默认 1，可选 1 到 10 |
| `nLevel` | N-back 的 N 值 | 由关卡映射，范围 1 到 4 |
| `roundCount` | 单局总轮数 | 20 到 40，随关卡增加 |
| `stimulusDuration` | 单个刺激展示时长 | 800ms 到 1200ms |
| `intervalDuration` | 两轮之间的间隔 | 300ms 到 600ms |
| `matchRate` | 匹配样本出现比例 | 25% 到 35% |
| `gridSize` | 网格规模 | 首期固定 3 × 3 |

### 4.5 难度曲线建议

| 关卡范围 | N 值 | 单局轮数 | 刺激展示时长 | 间隔时长 | 匹配比例 |
|---|---:|---:|---:|---:|---:|
| 1-2 | 1-back | 20 | 1200ms | 600ms | 30% |
| 3-5 | 2-back | 24 | 1100ms | 500ms | 30% |
| 6-8 | 3-back | 28 | 1000ms | 400ms | 30% |
| 9-10 | 4-back | 32 | 900ms | 350ms | 30% |

### 4.6 关卡映射函数建议

```typescript
function getNBackLevelConfig(startLevel: number): NBackLevelConfig {
  if (startLevel <= 2) {
    return { nLevel: 1, roundCount: 20, stimulusDuration: 1200, intervalDuration: 600, matchRate: 0.3 };
  }
  if (startLevel <= 5) {
    return { nLevel: 2, roundCount: 24, stimulusDuration: 1100, intervalDuration: 500, matchRate: 0.3 };
  }
  if (startLevel <= 8) {
    return { nLevel: 3, roundCount: 28, stimulusDuration: 1000, intervalDuration: 400, matchRate: 0.3 };
  }
  return { nLevel: 4, roundCount: 32, stimulusDuration: 900, intervalDuration: 350, matchRate: 0.3 };
}
```

### 4.7 刺激序列生成规则

1. 使用 3 × 3 网格，共 9 个位置，位置可用 `0-8` 表示。
2. 生成长度为 `roundCount` 的位置序列。
3. 对于第 `i` 轮：
   - 当 `i < nLevel` 时，不存在可比较的前 N 项，该轮不计入有效判断。
   - 当 `i >= nLevel` 时，按 `matchRate` 决定当前轮是否生成匹配项。
   - 若生成匹配项，则 `sequence[i] = sequence[i - nLevel]`。
   - 若生成非匹配项，则 `sequence[i]` 必须不等于 `sequence[i - nLevel]`。
4. 需要记录每轮是否为目标匹配轮，供判定使用。

### 4.8 交互流程

1. 用户进入 N-back 测试页。
2. 页面展示游戏说明、N-back 规则示例、历史最佳成绩、起始关卡选择器。
3. 用户选择起始关卡并点击「开始」。
4. 系统根据关卡生成 `nLevel`、轮数、展示时长和刺激序列。
5. 页面展示 3 × 3 网格。
6. 系统按节奏逐轮高亮一个格子。
7. 对于前 `nLevel` 轮，页面提示“观察中”，用户输入不计分。
8. 从第 `nLevel + 1` 轮开始，用户需要判断是否匹配。
9. 用户点击「匹配」或「不匹配」，也可使用键盘快捷键：
   - `Space`：匹配
   - `ArrowRight` 或 `N`：不匹配
10. 每轮结束后自动进入下一轮。
11. 全部轮次结束后进入结果页。

### 4.9 判定规则

| 场景 | 用户选择 | 系统判定 |
|---|---|---|
| 当前轮是匹配轮 | 匹配 | 命中，正确 |
| 当前轮是匹配轮 | 不匹配或未响应 | 漏报，错误 |
| 当前轮不是匹配轮 | 不匹配 | 正确拒绝，正确 |
| 当前轮不是匹配轮 | 匹配 | 误报，错误 |
| 前 `nLevel` 轮 | 任意输入 | 不计分，建议忽略输入 |

### 4.10 计分规则

首期建议使用简单可解释的计分方式：

```typescript
score = Math.round(accuracy * 1000 + nLevel * 100 + max(0, roundCount - missedCount * 2));
```

其中：

- `accuracy = correctCount / validRoundCount`
- `validRoundCount = roundCount - nLevel`
- `correctCount` 包括命中和正确拒绝
- `missedCount` 指有效轮次中未响应或超时的轮数

结果页需要展示以下指标：

| 指标 | 说明 |
|---|---|
| `accuracy` | 总正确率 |
| `hitRate` | 匹配轮中选择匹配的比例 |
| `falseAlarmRate` | 非匹配轮中误选匹配的比例 |
| `missCount` | 匹配轮中未选匹配的次数 |
| `bestNLevel` | 本轮完成的 N 值 |
| `score` | 综合分数 |

### 4.11 失败与结束机制

N-back 测试不建议采用“点错立即失败”的机制，而是采用**固定轮数完成后结算**。

1. 每局固定完成 `roundCount` 轮。
2. 中途答错不立即结束，只记录错误。
3. 用户可以点击「提前结束」，进入结果页。
4. 完成全部轮次后自动结算。
5. 若正确率达到通关阈值，则可提示用户挑战下一关。

建议通关阈值：

| 难度 | 通关正确率 |
|---|---:|
| 1-back | 70% |
| 2-back | 72% |
| 3-back | 75% |
| 4-back | 78% |

### 4.12 数据模型建议

```typescript
type NBackResponse = 'match' | 'non-match' | null;

type NBackTrial = {

  position: number;

type NBackLevelConfig = {
  nLevel: number;

type NBackState = {
  level: number;
  config: NBackLevelConfig;
  trials: NBackTrial[];

  status: GameStatus;
  startedAt?: number;
  currentTrialStartedAt?: number;
};

type NBackResult = {
  level: number;
  nLevel: number;
  score: number;
  accuracy: number;
  hitRate: number;
  falseAlarmRate: number;

```

### 4.13 实现提示

1. 建议新增 `NBackGame.tsx`、`logic.ts`、`types.ts`。
2. 刺激播放应使用定时器，但组件卸载时必须清理 `setTimeout` 或 `setInterval`。
3. 用户每轮最多只能提交一次判断，避免重复点击覆盖结果。
4. 若用户未在当前轮时间内作答，应记录为 `null`。
5. 前 `nLevel` 轮不计分，但仍应展示，用于建立记忆缓冲。
6. 结果页应同时展示正确率和得分，避免用户只看单一分数。
7. 移动端按钮需要足够大，「匹配」和「不匹配」建议固定在网格下方。
